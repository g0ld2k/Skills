# Reply Templates

Keep replies concise and specific. Avoid generic acknowledgements with no action.

## 1) Valid + Fixed

`Good catch. I updated <what changed> in <file> and added/updated <test or validation>.`

## 2) Valid + Alternative Implementation

`Thanks for calling this out. I addressed the issue with <chosen approach> because <reason>, which preserves <constraint/behavior>.`

## 3) Partial Validity

`You're right about <valid part>. I adjusted <change>, and kept <unchanged part> because <reason>.`

## 4) Invalid / Not Applicable

`I reviewed this path and don't see the issue in the current branch state. If you're seeing a specific case, can you share a repro or file/line?`

## 5) Needs Product/Design Decision

`I can implement this either as <option A> or <option B>. Which direction do you prefer for this PR?`

## Signature Guidance

Use a signature only if the user/repo convention asks for it.
